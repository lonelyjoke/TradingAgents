# TradingAgents v4 A股投研模块说明

v4 在 v3 的行情、技术面、估值、财务质量和三张表基础上，新增四个更接近真实投研流程的模块：

1. 事件层：抓取并分析近期公告、新闻、政策事件。
2. 行业对比层：把目标公司放到同行里比较。
3. 历史估值分位：判断当前估值处在历史高位、低位还是中间区间。
4. 大盘和板块风险层：判断市场和行业是否处在高风险位置，并识别“板块高位、个股低位”等背离。

这四个模块的共同目标是：让系统不要只看一家公司自己的数字，而是把它放进“事件、同行、历史、市场环境”里重新判断。

## 1. 事件层

事件层主要回答：

- 公司最近有没有重要公告？
- 最近有没有与公司或行业相关的新闻？
- 有没有政策、宏观或产业事件可能影响公司？
- 这些事件是短期情绪影响，还是长期基本面影响？

当前工具：

```text
get_company_events(ticker, curr_date, look_back_days=30)
```

数据尝试来源：

- Tushare 公司公告接口，例如上市公司公告。
- Tushare 重大新闻接口，例如财经新闻、行业新闻。
- Tushare 新闻联播/政策类接口，用于识别政策信号。

由于 Tushare 的部分资讯接口可能需要单独权限，系统会采用稳健策略：

- 能获取数据时，返回日期、标题、来源和内容摘要。
- 没有权限或接口不可用时，明确写出不可用原因。
- 不允许 AI 自己编造公告或新闻。

事件层在分析时要区分三类影响：

| 类型 | 例子 | 投研意义 |
| --- | --- | --- |
| 公司事件 | 业绩预告、分红、回购、减持、并购、问询函 | 直接影响公司价值或风险 |
| 行业事件 | 集采、价格战、行业补贴、供需变化 | 影响同行业估值和盈利预期 |
| 宏观政策 | 货币政策、财政政策、监管方向、产业政策 | 影响市场风险偏好和行业趋势 |

## 2. 行业对比层

行业对比层主要回答：

- 目标公司在同行里贵不贵？
- 盈利能力是否比同行更强？
- 增长、负债、股息是否有优势？
- 有没有同行看起来比目标公司更值得研究？

当前工具：

```text
get_peer_comparison(ticker, curr_date, peer_limit=12)
```

系统会先读取目标公司的 Tushare 行业分类，再在同一行业中选择市值较接近的同行做比较。

主要比较指标：

- PE TTM。
- PB。
- PS TTM。
- 股息率。
- ROE。
- 净利润同比增速。
- 资产负债率。
- 总市值。

系统还会给一个简单的 `v4_score`，用于快速筛查同行。这个分数不是投资结论，只是把估值、盈利质量、成长、负债和股东回报放在一起做初筛。

如果某个同行分数明显高于目标公司，AI 需要说明：

- 它为什么看起来更好？
- 是估值更低，还是质量更高？
- 这种优势是否可能只是短期数据造成的？
- 是否值得进一步单独跑一份完整投研？

## 3. 历史估值分位

历史估值分位主要回答：

- 这家公司现在的估值，在过去几年算高还是低？
- 当前低估值是机会，还是基本面变差后的合理折价？
- 当前高估值是否有足够成长和质量支撑？

当前工具：

```text
get_valuation_percentiles(ticker, curr_date, years=5)
```

主要观察：

- PE TTM 的历史分位。
- PB 的历史分位。
- PS TTM 的历史分位。
- 股息率的历史分位。

举例：

```text
PE TTM 当前处于过去 5 年 18% 分位
```

这通常说明估值相对历史偏低，但不能直接等于“便宜”。还要继续问：

- ROE 有没有下降？
- 利润增速有没有放缓？
- 现金流有没有恶化？
- 是否有行业政策或公司事件压低估值？

对价值投资来说，历史估值分位的正确用法是寻找问题，而不是直接给答案。

## 4. 大盘和板块风险层

这是 v4 新增的重要风险模块。它主要回答：

- 当前大盘是否处在高位或高估值区域？
- 当前板块是否整体偏贵、交易拥挤或风险偏高？
- 如果板块高位但目标个股低位，原因是什么？
- 如果大盘和板块都高风险，目标公司的安全边际是否足够？

当前工具：

```text
get_market_sector_risk(ticker, curr_date, look_back_days=120, years=5)
```

### 4.1 大盘风险怎么判断

系统会观察主要指数：

- 上证指数。
- 沪深300。
- 中证500。
- 创业板指。

主要指标：

- 近一段时间涨跌幅。
- 距离阶段高点的回撤。
- 当前点位在观察窗口里的位置。
- 指数 PE TTM 历史分位。
- 指数 PB 历史分位。

如果指数价格位置很高，同时估值分位也很高，系统会提示市场风险偏高。

### 4.2 板块风险怎么判断

系统会读取目标公司的 Tushare 行业分类，再把全市场同一交易日的股票按行业分组，计算行业中位数：

- 行业中位 PE TTM。
- 行业中位 PB。
- 行业中位 PS TTM。
- 行业中位股息率。
- 行业中位换手率。
- 行业中位市值。

然后把目标行业放到全市场所有行业中比较：

- 行业 PE 分位高，说明板块整体估值可能偏高。
- 行业 PB 分位高，说明市场对该板块资产或盈利预期较强。
- 行业换手率分位高，说明交易热度可能偏高。

### 4.3 如何看“板块高位、个股低位”

这是一个非常值得深挖的信号，但不能简单理解为机会。

如果板块高估值、高热度，而目标个股估值低于板块中位数，可能有两种完全不同的解释：

| 可能解释 | 含义 | 后续要查什么 |
| --- | --- | --- |
| 被错杀 | 公司基本面尚可，但市场暂时忽视 | 财务质量、事件影响、同行差异 |
| 有瑕疵 | 公司有同行没有的问题，所以估值低 | 公告、现金流、负债、治理、增长放缓 |

因此，AI 在报告里必须明确写：

- 这是潜在机会，还是潜在风险？
- 低估值来自市场误判，还是公司自身问题？
- 哪些数据支持进一步研究？
- 哪些风险会推翻这个机会假设？

## 5. v4 分析时的推荐顺序

对单只 A股标的，建议按下面顺序分析：

1. 先看公司基础信息和财务质量。
2. 再看历史估值分位，判断当前估值区间。
3. 再看同行对比，判断是否有更好的替代标的。
4. 再看事件层，判断近期公告、新闻、政策是否改变预期。
5. 最后看大盘和板块风险，判断当前是否适合提高仓位或只适合观察。

这个顺序的好处是：先看公司，再看价格，再看同行，再看事件，最后看环境。

## 6. 如何辩证使用 v4 输出

v4 比 v3 更全面，但仍然不是自动投资系统。

需要特别注意：

- 资讯接口可能受 Tushare 权限限制。
- 行业分类来自 Tushare，未必完全等同于基金经理眼中的真实可比公司。
- 简单同行分数只是筛查工具，不是最终评级。
- 历史估值分位不能脱离基本面变化。
- 大盘和板块风险只能帮助控制仓位和预期，不能单独决定买卖。

一句话：v4 的价值不是替你做决定，而是帮你把“公司、价格、同行、事件、市场环境”同时摆在桌面上，让你的投资判断更有结构。
