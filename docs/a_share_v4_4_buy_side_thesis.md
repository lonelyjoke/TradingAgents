# TradingAgents v4.4 买方下注型投研框架

v4.4 的目标，是把系统从“静态数据审查型报告”推进到“买方下注型投研框架”。

它不是鼓励 AI 脱离数据讲故事，而是要求系统在不完整信息中形成有边界的观点：

```text
用数据确定边界，
用产业逻辑形成推演，
用概率和赔率表达观点，
用仓位和证伪信号管理不确定性。
```

## 1. 为什么要做 v4.4

v4.3 已经解决了很多问题：

- 不轻易编产品价格；
- 能识别研究缺口；
- 能在缺微观数据时做供需替代分析；
- 能结合大盘、板块和估值分位校准评级；
- 能给止盈区间和进场观察区间。

但 v4.3 仍然偏保守。

当多空证据冲突、核心微观数据缺失，或者财报数据只能说明过去时，系统容易给出“持有”。这符合审慎原则，但不够像买方研究。

买方分析师真正要回答的是：

```text
如果我要看多，这只股票核心赌什么？
这个景气预期是否可能兑现？
市场有没有低估或高估这个变量？
这笔交易的胜率和赔率是否值得下注？
如果判断错了，什么信号会最先出现？
```

## 2. v4.4 的核心变化

v4.4 强制最终报告围绕几个问题展开。

### Core Bet / 核心下注主线

先说清楚：

```text
买这只股票，本质上是在下注哪个未来变量？
```

例子：

- 招商轮船：赌干散货景气和成本改善能否抵消油轮疲弱，并带来利润预期差；
- 巨化股份：赌制冷剂配额约束、产品价格韧性和价差修复；
- 紫金矿业：赌铜金价格中枢、矿山放量和成本控制；
- 银行股：赌低估值修复、分红吸引力和资产质量稳定。

如果说不清核心下注主线，系统就不应该硬给 Buy。

### Boom-Bust Expectation / 景气预期判断

判断这个主线是否可能兑现。

不能只说“行业景气改善”，而要结合：

- 宏观背景；
- 行业周期；
- 上游成本；
- 下游需求；
- 库存或开工率；
- 产品价格或代理价格；
- 运价或货量；
- 公司收入和利润结构；
- 同行业对比；
- 财报和现金流验证。

核心问题是：

```text
行业景气能不能传导到这家公司利润里？
```

### Expectation Gap / 预期差

好公司不一定是好交易，关键看市场是否已经定价。

系统需要判断：

- 利好是否已经被股价反映；
- 估值是否已经处在高分位；
- 市场是否仍按旧逻辑定价；
- 业绩改善是刚开始，还是已经成为一致预期；
- 板块是否拥挤；
- 是否存在“数据已经改善，但市场还没重估”的情况。

### Probability And Payoff / 胜率与赔率

v4.4 不再简单数多空证据，而是看：

```text
胜率：核心主线兑现的概率有多大？
赔率：如果兑现，上行空间多大？如果错了，下行风险多大？
```

评级更接近下面的逻辑：

- Buy：核心主线清晰，胜率高，赔率好，催化明确；
- Overweight：方向偏正面，但仍有数据缺口或需要分批验证；
- Hold：没有清晰主线，预期差不强，或胜率赔率普通；
- Underweight：负面主线更强，但还没到必须卖出；
- Sell：核心逻辑恶化，且风险收益明显不划算。

### Catalyst Path / 催化路径

系统需要说明接下来什么会推动市场重估。

例如：

- 产品价格继续上涨；
- 运价指数继续改善；
- 季报验证利润弹性；
- 政策落地；
- 行业库存下降；
- 出口改善；
- 分红或回购；
- 重要公告或订单。

### Falsification Signals / 证伪信号

每个看多或看空观点都必须能被证伪。

例如：

- BDI/BCI 回落，干散货主线失效；
- R32/R125 价格或价差下行，制冷剂主线弱化；
- 铜价回落或矿山放量不及预期，资源股逻辑降级；
- 银行不良率或息差恶化，红利低估逻辑受损。

如果没有证伪信号，说明这不是研究框架，而是故事。

### Conviction And Position / 观点强度与仓位

v4.4 会把观点和仓位联系起来。

- 高置信度：核心证据较充分，可以更积极；
- 中等置信度：方向成立，但有数据缺口，适合分批或中等仓位；
- 低置信度：逻辑有吸引力，但需要更多验证，适合观察仓；
- 负面置信度：应减仓或回避。

这让系统不会只有“买/持有/卖”，而是更像真实组合管理。

## 3. 内容整合做了什么

v4.4 同时收敛了报告内容。

以前的问题是：

- 基本面报告容易列太多财务数据；
- 技术分析容易罗列很多指标；
- 新闻分析容易变成新闻摘要；
- 社媒/情绪容易变成泛泛描述；
- 最终报告被迫从大量材料里找重点。

v4.4 的要求是：

- 基本面报告围绕核心下注主线、关键证据、缺口和预期差；
- 技术分析最多选择 5 个关键指标，只回答趋势、支撑阻力、动量和风险；
- 新闻报告只保留真正影响主线的公告、政策、行业事件；
- 情绪报告只关注市场是否拥挤、叙事是否变化、是否存在预期差；
- 最终报告围绕胜率、赔率、催化和证伪信号组织。

## 4. 如何辩证看待 v4.4

v4.4 不是让系统更激进，而是让系统更像买方。

它允许系统在数据不完美时形成方向，但有几个约束：

- 不能把推演写成事实；
- 不能编造产品价格、库存、价差、政策细节；
- 必须说明证据强弱；
- 必须列出关键研究缺口；
- 必须给证伪信号；
- 必须让仓位匹配置信度。

一句话总结：

```text
v4.4 允许下注预期，但要求下注是有证据边界、有赔率判断、有催化路径、有证伪纪律的。
```

## 5. 使用者应该怎么看输出

看一份 v4.4 报告时，建议按下面顺序读：

1. 先看 `Core Bet`，判断这个下注主线你是否认可；
2. 再看 `Boom-Bust Expectation`，判断景气预期是否有现实基础；
3. 再看 `Expectation Gap`，判断市场是否已经交易过这个逻辑；
4. 再看 `Probability And Payoff`，判断胜率赔率是否值得；
5. 再看 `Falsification Signals`，确认什么情况出现要认错；
6. 最后看评级、仓位、止盈区间和进场观察区间。

如果你不同意核心下注主线，即使评级是 Buy，也不应该直接接受。

如果你认同核心下注主线，但报告列出的证据缺口很多，那更适合小仓位跟踪，而不是重仓。
