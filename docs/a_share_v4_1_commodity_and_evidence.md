# TradingAgents v4.1 商品价格与证据约束说明

v4.1 主要解决一个真实投研问题：AI 不能把没有来源的行业数字写成事实。

例如，分析巨化股份时，如果系统没有接入 R32、R125 的可靠报价数据，agent 就不能写：

```text
截至某日，R32 报价已经下跌 12%-15%。
```

它只能写：

```text
R32 价格走势是巨化股份的重要利润变量，但当前系统没有获取到可验证报价，需要人工核验。
```

## 1. 新增商品价格工具

新增工具：

```text
get_commodity_context(ticker, curr_date, look_back_days=90)
```

这个工具会根据股票代码尝试识别主力产品和关键原料，然后返回一张证据表。

当前支持两类数据路径：

| 类型 | 适用产品 | 数据方式 | 可信度 |
| --- | --- | --- | --- |
| 标准化期货 | 铜、黄金、碳酸锂、铝、螺纹钢、焦煤等 | Tushare 期货行情、仓单数据 | 较高 |
| 非标准化现货 | R32、R125、R134a、PVDF、萤石、氢氟酸等 | 白名单网页证据片段 | 需要人工复核 |

## 2. 为什么 R32/R125 不能直接当成硬数据

R32、R125 这类制冷剂不是标准交易所期货合约，不能像铜、铝、碳酸锂那样直接从 Tushare 期货接口拿到连续行情。

所以 v4.1 对它们采用更保守的方式：

- 只访问白名单网站。
- 抓取网页标题和价格相关片段。
- 不强行解析为精确价格。
- 不自动计算涨跌幅。
- 把结果标记为“网页证据，需要人工复核”。

这样做的目的不是降低分析能力，而是避免把网页片段误写成精确行情。

## 3. 当前内置的股票-产品映射

第一版先内置少量映射，后续可以继续扩展。

| 股票 | 主力产品/变量 | 数据方式 |
| --- | --- | --- |
| 600160.SH 巨化股份 | R32、R125、R134a、PVDF、萤石、氢氟酸 | 白名单网页证据 |
| 601899.SH 紫金矿业 | 铜、黄金 | Tushare 期货代理 |
| 002460.SZ 赣锋锂业 | 碳酸锂 | Tushare 期货代理 |
| 300750.SZ 宁德时代 | 碳酸锂成本代理 | Tushare 期货代理 |

如果一个股票没有映射，系统会尝试根据公司名称和行业做简单推断。推断结果只能当作线索，不能当成公司真实收入结构。

## 4. 证据约束补丁

v4.1 给多个 agent 都加了证据纪律：

- 基本面分析师。
- 新闻分析师。
- 多头研究员。
- 空头研究员。
- 研究经理。
- 交易员。
- 风险分析师。
- 组合经理。

核心规则：

```text
凡是具体数字、百分比、商品价格、库存、产能、配额、价差、日期型市场判断，都必须来自工具输出或已有分析报告。
如果没有证据，必须标记为“未验证关键假设”，不能当作事实。
```

## 5. 对投研结果的影响

v4.1 以后，agent 应该更少出现这种问题：

```text
R32 下跌 12%-15%
某商品库存下降 20%
某产品价差扩大 30%
某政策已经落地
```

除非这些数字已经出现在工具证据表里。

对于周期股、资源股、化工股，报告会更关注：

- 产品价格。
- 原材料价格。
- 库存或仓单。
- 产品-原料价差。
- 价格变化对毛利率和净利润的影响。

但如果系统没有可靠数据，它会把这些变量列为“待核验”，而不是编一个数字。

## 6. 后续可扩展方向

后续可以继续加入：

- 更多股票-产品映射。
- 百川盈孚、卓创资讯、隆众资讯、生意社等更稳定的数据源。
- 公司主营收入结构自动识别。
- 产品价格和原材料价格的价差曲线。
- 行业库存、开工率、产能利用率。
- 对商品价格敏感的利润弹性测算。

一句话：v4.1 的目标不是让 AI 说得更满，而是让它说得更有证据。
