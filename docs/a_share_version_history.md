# TradingAgents A股投研系统版本迭代记录

这份文档用于记录本项目从基础复现到 A股投研框架的功能演进。它不是代码接口文档，而是给使用者和后续开发者看的版本说明：每一版解决了什么问题，新增了哪些功能，还剩哪些边界。

## 总体定位

当前系统不是自动交易程序，也不是保证收益的选股器。它更像一个 AI 投研工作台：把行情、财务、估值、公告、同行、商品价格、航运指数等证据交给多个 agent，让它们围绕一个标的形成结构化分析。

使用时应始终记住：

- AI 输出是投研辅助，不是投资指令。
- 具体数字必须有数据来源。
- 没有证据的数据只能作为待核验假设。
- 最终结论需要人工复核。

## v1：基础复现版本

v1 的目标是先把原始 TradingAgents 项目在本地跑起来。

主要工作：

- 搭建 Python 虚拟环境。
- 安装项目依赖。
- 配置 DeepSeek API key。
- 让命令行程序可以正常启动。
- 跑通最基础的单标的分析流程。

这一版解决的问题：

- 确认项目可以在 Windows 本地运行。
- 确认 DeepSeek 可以作为模型后端使用。
- 理解原项目的多 agent 流程，包括市场分析师、基本面分析师、新闻分析师、多空辩论、交易员、风控和组合经理。

主要边界：

- 仍然偏原始项目逻辑。
- 对 A股支持不足。
- 依赖 yfinance 时容易限流。
- 对中文 A股投研语境适配较弱。

## v2：A股数据源切换与 Tushare 接入

v2 的目标是解决 A股行情数据不稳定的问题。

主要工作：

- 引入 Tushare。
- 新增 Tushare 初始化逻辑。
- 支持从 `.env` 读取 `TUSHARE_TOKEN` 和自定义 Tushare 网关地址。
- 对 `.SZ`、`.SH`、`.BJ` 这类 A股代码优先走 Tushare。
- 修复 yfinance 限流导致 A股分析无法继续的问题。

这一版解决的问题：

- A股日线行情获取更稳定。
- 不再主要依赖 yfinance 分析 A股。
- 可以用真实 A股代码跑出基础分析结果。

主要边界：

- 基本面和财务数据仍不够完整。
- 新闻、公告、行业事件还没有系统接入。
- 还没有同行对比和历史估值分位。

## v3：A股基本面与技术面增强版本

v3 的目标是把系统从“能跑 A股”升级为“能做基础 A股投研”。

新增数据：

- A股日线行情。
- 技术指标，包括均线、MACD、RSI、布林带、ATR、MFI 等。
- 公司基础信息。
- 每日估值和交易快照，包括 PE、PB、PS、股息率、换手率、市值等。
- 财务质量指标，包括 EPS、ROE、ROA、ROIC、毛利率、净利率、资产负债率、营收增速、净利润增速等。
- 利润表。
- 资产负债表。
- 现金流量表。
- 主要指数和申万行业快照。

新增能力：

- 基本面分析师可以看到更完整的公司画像。
- 市场分析师可以基于 Tushare 日线数据计算技术指标。
- 财务分析不再只停留在单个估值数字，而是能看盈利能力、成长性、现金流和负债。

这一版解决的问题：

- A股分析从“行情分析”扩展到“行情 + 财务 + 估值”。
- 对价值投资者更友好。
- 可以初步判断公司质量、估值状态和市场位置。

主要边界：

- 还没有近期公告、新闻和政策事件层。
- 同行业横向比较不足。
- 当前估值缺少历史分位参考。
- 对周期品、资源品、航运等行业的核心景气变量覆盖不足。

## v4：事件、同行、估值分位、市场/板块风险版本

v4 的目标是把单公司分析放进更真实的投研环境中。

新增模块：

### 事件层

新增 `get_company_events`。

功能：

- 获取公司公告。
- 尝试获取公司和行业相关新闻。
- 尝试获取政策或宏观事件。
- 对公告做初步分类，例如业绩/财报、分红/权益、回购/增减持、融资/债券、并购/投资、治理/人事、股东大会、关联交易、监管/法律等。

优化点：

- 如果新闻或政策接口没有权限，会明确提示。
- 不允许 agent 编造新闻。
- 公告作为更硬的证据优先展示。

### 行业对比层

新增 `get_peer_comparison`。

功能：

- 根据 Tushare 行业分类寻找同行。
- 选择市值较接近的同行进行横向比较。
- 比较 PE、PB、PS、股息率、ROE、净利润增速、资产负债率、市值等。
- 生成简单的 `v4_score`，用于初步筛查同行。

优化点：

- agent 可以判断是否存在比目标公司更值得研究的同行。
- 防止只看单家公司而忽略行业内更优选择。

### 历史估值分位

新增 `get_valuation_percentiles`。

功能：

- 计算 PE TTM、PB、PS TTM、股息率在历史区间的位置。
- 判断当前估值处于历史高位、低位还是中间区域。

优化点：

- 帮助区分“真的便宜”和“因为基本面恶化才便宜”。
- 让估值判断更有历史参照。

### 大盘和板块风险

新增 `get_market_sector_risk`。

功能：

- 观察上证指数、沪深300、中证500、创业板指。
- 判断主要指数的价格位置、回撤、估值分位。
- 计算目标行业在全市场行业中的估值和换手率位置。
- 识别“板块高位但个股低位”这类背离。

优化点：

- 帮助判断市场和板块是否处于高风险区域。
- 如果板块高估但个股低估，会提示需要深挖：是错杀，还是公司自身瑕疵。

主要边界：

- 事件层受 Tushare 权限影响较大。
- 同行分类依赖 Tushare 行业标签，不能完全替代人工行业理解。
- 市场/板块风险是风险提示，不是买卖信号。

## v4.1：商品价格与证据约束版本

v4.1 的目标是解决一个重要问题：AI 不能把没有来源的行业数字写成事实。

背景问题：

在分析周期股时，agent 可能会生成类似这样的句子：

```text
截至某日，R32 报价从高点下滑 12%-15%。
```

如果系统没有真实抓到 R32 报价，这类句子就是未验证数据，甚至可能是幻觉。

新增模块：

### 商品价格/产业品证据层

新增 `get_commodity_context`。

功能：

- 根据股票代码识别主力产品和关键原料。
- 对有期货的商品，使用 Tushare 期货行情做代理。
- 对没有标准期货的产品，使用白名单网页抓取证据片段。
- 输出产品、数据来源、最新价格、日期、区间变化、库存/仓单、证据状态。

当前内置映射：

- 巨化股份：R32、R125、R134a、PVDF、萤石、氢氟酸。
- 紫金矿业：铜、黄金。
- 赣锋锂业：碳酸锂。
- 宁德时代：碳酸锂成本代理。

优化点：

- 对铜、黄金、碳酸锂等标准化商品，可以使用 Tushare 期货数据。
- 对 R32、R125 等非标准化产品，只提供网页证据，不自动编造精确涨跌幅。
- 没有可靠数据时，会明确标注“不可作为事实”。

### 证据约束补丁

新增统一证据纪律。

约束对象：

- 基本面分析师。
- 新闻分析师。
- 多头研究员。
- 空头研究员。
- 研究经理。
- 交易员。
- 风险分析师。
- 组合经理。

核心规则：

```text
凡是具体数字、百分比、商品价格、库存、产能、配额、价差、日期型市场判断，
必须来自工具输出或已有分析报告。
如果没有证据，必须标记为“未验证关键假设”，不能当作事实。
```

这一版解决的问题：

- 降低 AI 编造行业价格、库存、价差的风险。
- 让周期股、资源股、化工股的分析更接近真实投研。
- 帮助最终报告区分“已验证证据”和“待核验假设”。

主要边界：

- 白名单网页证据不等于完整数据库。
- 非标准化现货价格仍需要人工复核。
- 商品到公司利润的弹性测算还没有完整模型。

## v4.3：大盘情绪校准与操作区间版本

v4.3 的目标是让系统不只给出静态评级，还能结合市场环境和标的特性，给出更贴近投资操作的区间建议。

新增模块：

### 大盘情绪与评级校准

新增 `get_market_timing_context`。

功能：

- 观察上证指数、沪深300、中证500、创业板指。
- 综合指数位置、回撤、PE/PB 历史分位和换手率。
- 判断当前市场属于极度悲观、偏熊、均衡、偏牛但风险升高、极度乐观等状态。
- 给出评级校准建议。

核心原则：

- 牛市中可以适度积极，但不能机械追高。
- 熊市中应更保守，但极度悲观低估时可以适度逆向。
- 极度乐观时应提高止盈和减仓纪律。
- 极度悲观时应重点寻找财务稳健、估值低、催化清晰的标的。
- 周期股、商品股、航运股、高 beta 股票要结合自身行业变量，不能只看大盘情绪。

### 操作区间

最终报告新增或强化：

- `Market Regime Adjustment`：大盘和板块情绪如何影响评级。
- `Profit-Taking / Trimming Range`：偏多时的止盈或减仓区间。
- `Entry / Re-entry Watch Range`：偏空时的进场观察区间。
- `Unverified Key Assumptions`：对结论重要但缺少证据的数据假设。

这一版解决的问题：

- 避免评级过于静态。
- 让同一只股票在不同市场环境下有不同仓位纪律。
- 偏多时不只说看好，还提示涨到哪里该考虑减仓。
- 偏空时不只说回避，还提示跌到哪里值得重新观察。

主要边界：

- 区间不是精确预测，只是基于证据的操作纪律。
- 如果缺少技术位、估值位或周期变量证据，系统应给情景区间或标记待核验。
- 市场情绪只用于校准，不替代公司基本面和行业研究。

## v4.4：买方下注型投研框架

v4.4 的目标是让系统更像买方分析师，而不是只做静态数据审查。

新增和优化：

- 新增 `Core Bet`：先说明如果看多或看空，核心下注变量是什么。
- 新增 `Boom-Bust Expectation`：判断景气预期是否可能兑现。
- 新增 `Expectation Gap`：判断市场是否已经充分定价。
- 新增 `Probability And Payoff`：用胜率和赔率解释评级，而不是简单数多空证据。
- 新增 `Catalyst Path`：说明后续什么事件或数据可能推动重估。
- 新增 `Falsification Signals`：说明什么信号出现代表逻辑错了。
- 新增 `Conviction And Position`：把观点强度和仓位建议绑定。

报告收敛：

- 基本面分析从“全面罗列”改为“围绕交易主线组织证据”。
- 技术分析最多选择 5 个关键指标，减少低价值指标堆叠。
- 新闻分析从泛泛摘要改为催化和风险事件备忘录。
- 情绪分析从长篇描述改为观察拥挤度、叙事变化和预期差。
- 最终报告优先回答：赌什么、为什么可能兑现、市场是否定价、胜率赔率如何、错了看什么。

核心原则：

- 可以基于数据和产业逻辑做有边界的推演。
- 不能把推演写成事实。
- 数据缺失不必然等于 Hold。
- Hold 应该表示没有清晰可交易主线、预期差不足，或胜率赔率普通。
- Overweight 可以用于“方向偏正面但仍需验证”的买方观点。
- 仓位必须匹配证据强度和置信度。

对大多数 A股标的，建议至少选择：

```text
Market Analyst
Fundamentals Analyst
News Analyst
```

对于资源、化工、航运等周期股，尤其建议开启基本面分析师，因为 v4.1 和 v4.2 的商品/航运工具主要接在基本面分析链路里。

启动命令：

```powershell
cd C:\Users\zhanc\Documents\GitHub\TradingAgents
.\.venv\Scripts\tradingagents.exe
```

## v4.4.1：景气-估值公允校准补丁

v4.4.1 的目标是避免系统机械偏向某一种投资风格。

新增 `Fair cycle-valuation calibration` 规则：

- 低估值低景气不自动看空；
- 高景气高估值不自动看多；
- 低估值高景气要检查盈利是否可持续；
- 高估值低景气要检查是否有明确拐点或稀缺性；
- 最终评级应由胜率、赔率、持续性、资产负债表韧性和预期差共同决定。

最终报告新增：

- `Cycle-Valuation Assessment`

这个字段会说明标的处在什么组合中：

- 低估值 / 低景气；
- 高估值 / 高景气；
- 低估值 / 高景气；
- 高估值 / 低景气。

系统会据此判断：便宜是不是机会，高景气是不是陷阱，低景气是否已被充分定价，高估值是否仍能被成长消化。

## v4.5：投研观点验证系统

v4.5 的目标是把主观投研框架放到真实行情里检验。

新增：

- `tradingagents/evaluation/research_validator.py`
- `scripts/validate_research_reports.py`
- `docs/a_share_v4_5_research_validation.md`

第一版验证内容：

- 读取 `reports` 目录下已经生成的报告；
- 解析标的、报告日期、最终评级、核心下注主线、观点强度；
- 用 Tushare 获取报告生成后的真实行情；
- 默认使用报告后下一个交易日收盘价作为入场价；
- 计算 20/60/120 个交易日收益；
- 计算相对沪深300的超额收益；
- 计算最大回撤和最大浮盈；
- 按评级判断命中情况；
- 输出详细 CSV 和汇总 CSV。

运行方式：

```powershell
.\.venv\Scripts\python.exe scripts\validate_research_reports.py --reports reports --horizons 20 60 120
```

设计原则：

- 只使用真实 Tushare 数据；
- 数据不足就标记 `insufficient_future_data`；
- 拉不到价格就标记 `price_unavailable`；
- 不用样本数据替代真实结果；
- 先验证观点，再做组合回测。

## v4.5.1：买方财报雷达与前瞻会计信号

v4.5.1 的目标，是把基本面分析从“看利润表结果”推进到“从财报会计项里寻找未来业绩线索”。

这次增强的出发点是：财报虽然滞后，但它是最可靠的事实底稿。优秀买方分析师不能只看 PE/PB、净利润和毛利率，也要从合同负债、应收、存货、预付款、应付、商誉、现金流和资本开支等项目中，判断企业未来收入、利润质量、周期承压能力和股东回报。

新增内容：

- 新增文档 `docs/a_share_buy_side_accounting_radar.md`，系统整理“买方财报雷达”。
- 基本面 Agent 新增 `Buy-side accounting radar` 分析纪律。
- `get_fundamentals` 的派生财务指标中补充前瞻会计信号。
- 资产负债表数据读取新增合同负债、预收款、合同资产、应收票据、其他应收、预付款、应付票据、其他应付、流动资产、流动负债等字段。
- 派生计算新增合同负债+预收款/收入、合同负债环比变化、应收/收入、存货/收入、预付款/收入、应付/营业成本、商誉/资产、净现金、营运资本等指标。

买方财报雷达固定围绕五个问题：

1. 收入真实吗？
2. 利润质量硬不硬？
3. 未来业绩有没有会计线索？
4. 资产负债表能不能扛周期？
5. 股东回报是否匹配？

具体改进：

- 合同负债和预收款不再只是普通负债项目，而是被视为潜在订单、预收收入和未来收入可见度线索。
- 应收账款、应收票据、合同资产会用于判断收入含金量和回款风险。
- 存货、预付款和应付账款会结合行业景气判断，是主动备货、需求压力、锁定原材料，还是产业链议价能力变化。
- 商誉、减值、净现金、营运资本会用于判断公司是否存在资产减值风险，以及是否能扛过行业低谷。
- Agent 被要求不要机械解读会计项。例如，合同负债增加对工程、软件、设备、白酒等公司可能偏正面，但对现货型周期品公司意义可能有限；存货上升可能是备货，也可能是滞销，必须结合行业价格和供需判断。

分析边界：

- 财报是事实锚，不是完整结论。
- 派生指标是系统根据财报科目计算出来的，不等于公司直接披露的指标。
- 一季报、中报、三季报的利润表和现金流项目可能是年初至报告期累计口径，不能简单当成单季度口径。
- 会计项只能提供线索，最终仍需要结合行业景气、产品价格、订单、产能、估值和市场预期差判断。

这个版本让系统更像买方分析师读财报：先用财报确认经营现实，再用行业景气和估值赔率判断是否值得下注。

## v4.6：叙事催化、政策上下文与财报深读增强版

v4.6 的目标，是把系统从“能看到更多资料”继续推进到“能判断哪些资料会改变投资结论”。  
这次升级的直接触发，是在复盘金风科技时发现：系统虽然已经能看到一部分基本面信息，但仍会遗漏已经落地的股东事件、一级投资能力、互动平台披露出的经营线索，以及财报中真正可能改变估值框架的第二增长极。

这一版不再满足于让 agent 罗列信息，而是要求它回答更接近买方研究的问题：

1. 这条信息是否真实、是否已经落地？
2. 它会影响收入、利润率、现金流、估值，还是只影响市场想象力？
3. 它应当进入多头逻辑、空头反驳，还是只保留为低权重叙事期权？
4. 如果财报已经给出线索，后续还需要什么证据才能完成验证？

### 新增模块

#### 1. 叙事催化与题材映射层

新增主题研究链路，用于识别 A 股市场中会影响预期差的叙事变量：

- 接入概念映射，识别公司所处的主题池和市场标签。
- 允许互动平台、媒体联想、概念映射进入多头讨论，但只给予有限的“想象力溢价”，不能直接替代盈利兑现。
- 将题材证据区分为不同可信层级，并加入交叉验证逻辑，避免单一概念或单条问答被过度拔高。
- 支持从财报中挖掘一级投资能力、优质参股项目和潜在资产重估线索，再判断其是否具备真实估值意义。

设计原则：

- A 股会给故事定价，但系统不能把故事写成事实。
- 低可信题材可以进入讨论，但只能作为期权，不可直接抬升核心盈利预测。
- 一旦存在财报、公告或公司正式回复支撑，叙事可以从“想象力”升级为“可验证催化”。

#### 2. 投资者互动平台层

新增：

- `tradingagents/dataflows/investor_interaction_research.py`

当前已接入：

- 深交所互动易。
- 上证 e 互动。

能力包括：

- 通过官方接口抓取投资者问答。
- 区分未回复、模板式回复与实质性回复。
- 从问答中提取经营进展、第二曲线、产能建设、订单、股东人数、市值管理、资本开支等可能影响预期的线索。
- 将有价值的互动信息送入多头、空头和组合经理的讨论框架，而不是仅停留在背景资料。

典型用途：

- 公司在互动平台首次承认某项业务已进入建设、试生产、签约或落地阶段。
- 公司对市场误解进行澄清，例如回购、减持、订单、在手项目、客户关系等。
- 投资者问题本身暴露市场正在聚焦的边际变量。

边界：

- 互动平台回复属于公司口径，不等于财报兑现。
- 未经财报、公告或后续经营数据验证时，最多只能提高研究优先级，不能直接当作盈利事实。

#### 3. 政策规划与行业文件层

新增：

- `tradingagents/dataflows/policy_research.py`

能力包括：

- 接入国家级规划、中央政策文件与行业规划文件。
- 依据公司所属行业，路由相关政策文档。
- 区分“国家战略方向”“行业需求斜率”“公司自身受益”三个层次。
- 将政策映射为多头和空头都可使用的证据：
  - 多头判断政策是否提高长期 TAM、需求确定性与产业优先级。
  - 空头判断政策是否只扩行业规模，却不改善公司利润率、竞争格局或资本回报。

设计原则：

- 政策文件是强方向证据，不是直接利润证明。
- 行业文件的解释力高于泛化的宏观口号。
- 只有当政策能穿透到订单、价格、产能利用率或估值锚时，才应真正影响核心判断。

#### 4. 股东事件生命周期修正

新增对股东减持、回购、筹码结构和历史事件状态的更细致处理：

- 已经执行完毕的减持，不再被机械写成“未来压制因素”。
- 股东事件需要区分：
  - 尚未发生。
  - 进行中。
  - 已完成。
  - 虽已完成但留下筹码结构后遗症。
- 多头和空头需要分别讨论“事件本身”与“事件之后留下的股东结构变化”，避免把历史包袱误写成未来供给。

#### 5. 财报深读系统

新增并系统化了财报阅读框架：

- `Filing Reading Coverage Audit`
- `Paragraph Reading Pack`
- `Industry Reading Pack`
- `Statement Table Reading Pack`
- `Filing Note Reading Pack`
- `Financial Relationship Reading Pack`
- `Business Model Map`
- `Growth Vector Map`
- `Core Discussion Promotion Queue`
- `Unanswered Filing Questions`

这意味着系统读财报时，不再只看几个财务指标，而是按十个层次推进：

1. 先确认财报是否真正读到了。
2. 再读正文经营叙述。
3. 再按行业框架找关键问题。
4. 再读利润表、资产负债表、现金流量表。
5. 再读附注中的客户、担保、诉讼、减值、关联方和投资项目。
6. 再做财务关系推理。
7. 再识别商业模式与第二增长极。
8. 再判断增长向量是否进入兑现阶段。
9. 最后把真正值得影响投资判断的信息提升到核心讨论。
10. 对没有回答清楚的问题显式留痕，而不是静默遗漏。

### 财务关系推理的新增重点

这一版开始让系统从“看到单个数字”升级为“理解数字之间的关系”：

- 收入增长是否带来毛利率改善。
- 利润增长是否伴随经营现金流恶化。
- 合同负债增加是否真的对应更高质量订单，还是仅仅反映预收比例变化。
- 毛利率改善是否真正传导到营业利润。
- 资本开支、存货、预付款和应收账款是否共同说明企业正在扩张，还是在透支现金。
- 收入、利润、现金流、资产负债表是否共同指向“高质量成长”，还是只是一段表面繁荣。

这一步非常关键。  
买方分析的本质，不是背财报科目，而是看懂报表之间是否彼此讲得通。

### 这一版重点解决的问题

- 已经落地的减持事件被继续误判为未来压制。
- 一级投资能力、优质参股项目和资产重估线索未进入多头框架。
- 互动平台和题材信息被采集后，没有进入真正的研究辩论。
- 政策和行业规划文件没有进入长期需求空间判断。
- 财报中对第二增长极、长协订单、业务转型和兑现阶段的描述没有被充分提升。
- 信息层已经拿到线索，但分析层仍然围绕旧问题重复辩论。
- 系统会摘取财报语句，却没有系统回答“这条语句为什么重要”。

### 分析层改造

为了避免“采集到了但没讨论”，这一版把信息流重新接到了结论层：

- 基本面分析师需要先完成财报阅读覆盖检查，再形成经营判断。
- 多头研究员必须消费能够提升估值、扩展 TAM 或证明第二增长极的核心材料。
- 空头研究员必须判断这些材料是否已兑现、是否只停留在叙事、是否会被竞争或现金流问题抵消。
- 组合经理需要看到：
  - 哪些是已经证实的事实。
  - 哪些是高可信催化。
  - 哪些只是低权重叙事期权。
  - 哪些仍属于未回答问题。

这使系统从“信息搬运”更接近“投资委员会讨论”。

### 当前能力边界

这版已经形成较完整的买方研究骨架，但还不能被视为成熟终局：

- PDF 文本、表格和附注理解仍然带有启发式特征，还不是完全结构化阅读。
- 不同行业之间的财报重点已经开始分流，但离优秀行业分析师的深度还存在距离。
- 跨期管理层承诺兑现、长期项目跟踪、异常解释链条仍需继续增强。
- 低可信叙事虽然已被纳入，但权重控制仍需要依赖后续实盘回看不断校准。
- 旧版本生成的历史研报，不应被拿来代表当前版本能力；需要用新版重新跑样本做正式验收。
