# TradingAgents v3 A股投研框架说明

这份文档写给第一次接触本项目的人。你可以把 v3 理解成一个“AI 投研会议室”：它不是传统意义上的自动交易程序，也不是一个保证赚钱的选股器，而是把 Tushare 的 A股数据交给多个 AI 角色，让它们围绕同一个标的做技术面、基本面、风险和组合层面的讨论，最后给出一份可读的投研结论。

## 1. v3 版本解决什么问题

v3 的目标是让 TradingAgents 更适合分析 A股，尤其是价值投资和中短期观察结合的场景。

它主要做三件事：

1. 自动识别 A股代码，例如 `000001.SZ`、`600519.SH`、`920001.BJ`。
2. 对 A股优先使用 Tushare 数据，而不是默认依赖 yfinance。
3. 给 AI 分析师提供更完整的 A股资料，包括行情、技术指标、估值、财务质量、三张表、大盘和行业环境。

它不做三件事：

1. 不自动下单。
2. 不保证买卖建议正确。
3. 不等同于严谨的量化回测系统。

更准确地说，它是一个“AI 投研辅助系统”。它适合帮你快速整理线索、发现矛盾、形成检查清单，但最终判断仍然需要人来做。

## 2. 运行前需要准备什么

你需要在项目根目录准备 `.env` 文件，里面至少包含：

```text
DEEPSEEK_API_KEY=你的 DeepSeek key
TUSHARE_TOKEN=你的 Tushare token
TUSHARE_HTTP_URL=你的 Tushare 网关地址
```

注意：

- 变量名必须使用下划线，例如 `TUSHARE_TOKEN`，不要写成 `TUSHARE-TOKEN`。
- 不要把 `.env` 文件上传到公开仓库。
- 文档和代码里都不应该保存真实 token。

启动命令：

```powershell
cd C:\Users\zhanc\Documents\GitHub\TradingAgents
.\.venv\Scripts\tradingagents.exe
```

推荐第一次使用 v3 时这样选：

- 标的：先用一个熟悉的大股票，例如 `000001.SZ` 或 `600519.SH`。
- 分析师：先选 `Market Analyst` 和 `Fundamentals Analyst`。
- 研究深度：先选 `Shallow`。
- 模型：使用 DeepSeek。
- 输出语言：可以要求中文。

等这套轻量流程稳定后，再增加更多分析师和更深的研究深度。

## 3. v3 获取了哪些数据

v3 目前主要从 Tushare 获取以下几类数据。

### 3.1 日线行情

接口来源：`daily`

主要字段：

- 交易日期
- 开盘价
- 最高价
- 最低价
- 收盘价
- 成交量
- 成交额
- 涨跌幅

这些数据主要给市场分析师使用。它用来判断股价最近是上涨、下跌、横盘，成交量有没有放大，价格有没有接近关键位置。

### 3.2 技术指标

接口来源：先取 Tushare 日线行情，再由本地 `stockstats` 计算。

当前支持的典型指标包括：

- 50日均线：观察中期趋势。
- 200日均线：观察长期趋势。
- 10日 EMA：观察短期动量。
- MACD、MACD signal、MACD histogram：观察趋势动能。
- RSI：观察是否过热或过冷。
- 布林带：观察价格波动区间。
- ATR：观察波动率，可辅助理解止损空间。
- VWMA：成交量加权均线。
- MFI：结合价格和成交量观察资金流强弱。

这些指标不是“买卖按钮”，更像体检指标。单个指标不能直接决定交易，应该和趋势、估值、基本面、市场环境一起看。

### 3.3 公司基础信息

接口来源：`stock_basic`

主要字段：

- 股票代码
- 公司名称
- 所在地区
- 所属行业
- 市场板块
- 交易所
- 上市日期

这部分用于确认“我们到底在分析谁”。例如，同样是银行股、白酒股、半导体股，它们适用的估值逻辑和风险点不同。

### 3.4 估值和交易快照

接口来源：`daily_basic`

主要字段：

- 收盘价
- 市盈率 PE
- 滚动市盈率 PE TTM
- 市净率 PB
- 市销率 PS
- 滚动市销率 PS TTM
- 股息率
- 换手率
- 量比
- 总市值
- 流通市值

这些数据主要用来回答：

- 这家公司现在贵不贵？
- 市场交易热度高不高？
- 它是大盘股、中盘股，还是小盘股？
- 股息率是否值得关注？

对价值投资来说，这一层很重要，但它还不够。低 PE 不一定便宜，高 PE 也不一定泡沫，必须结合盈利质量、成长性、行业周期和资产负债表一起判断。

### 3.5 财务质量指标

接口来源：`fina_indicator`

主要字段：

- EPS：每股收益。
- BPS：每股净资产。
- OCFPS：每股经营现金流。
- 毛利率。
- 净利率。
- ROE。
- ROA。
- ROIC。
- 资产负债率。
- 流动比率。
- 速动比率。
- 营收同比增速。
- 净利润同比增速。

这部分是 v3 基本面分析的核心。它帮助 AI 判断：

- 公司赚钱能力强不强。
- 利润率有没有变好或变差。
- ROE 是否稳定。
- 增长来自收入增长，还是只来自费用、投资收益或周期波动。
- 债务压力是否偏高。
- 现金流和利润是否匹配。

### 3.6 三张财务报表

v3 会按最近报告期拉取三张表的关键字段。

利润表，接口来源：`income`

- 营业总收入。
- 营业收入。
- 营业总成本。
- 营业利润。
- 利润总额。
- 净利润。
- 归母净利润。
- 销售费用。
- 管理费用。
- 财务费用。
- 研发费用。
- EBIT。
- EBITDA。

资产负债表，接口来源：`balancesheet`

- 总资产。
- 总负债。
- 股东权益。
- 货币资金。
- 应收账款。
- 存货。
- 固定资产。
- 商誉。
- 短期借款。
- 长期借款。
- 应付债券。
- 应付账款。

现金流量表，接口来源：`cashflow`

- 净利润。
- 销售商品、提供劳务收到的现金。
- 经营活动现金流净额。
- 投资活动现金流净额。
- 筹资活动现金流净额。
- 期初现金及现金等价物。
- 期末现金及现金等价物。

三张表的意义可以这样理解：

- 利润表：公司账面上赚了多少钱。
- 资产负债表：公司家底如何，债务压力大不大。
- 现金流量表：利润有没有变成真金白银。

价值投资里，现金流和资产负债表通常比短期股价更重要。

### 3.7 大盘和行业环境

接口来源：`index_daily` 和 `sw_daily`

当前会观察的主要指数包括：

- 上证指数。
- 沪深300。
- 中证500。
- 创业板指。

行业快照来自申万行业数据，用于观察当天行业层面的涨跌状态。

这部分用于回答：

- 个股下跌是自身问题，还是大盘整体较弱？
- 个股上涨是公司独立走强，还是行业整体普涨？
- 当前市场风格偏大盘、偏成长、偏周期，还是偏防守？

## 4. v3 如何分析一个标的

以 `600519.SH` 为例，系统大致会经历下面这条链路。

### 4.1 用户输入

你输入：

- 股票代码，例如 `600519.SH`。
- 分析日期，例如当天或某个历史日期。
- 选择哪些 AI 分析师。
- 选择研究深度。
- 选择模型，例如 DeepSeek。

### 4.2 系统判断这是 A股

只要代码以 `.SZ`、`.SH` 或 `.BJ` 结尾，系统就会优先走 Tushare 数据通道。

这一步很重要。因为默认国际数据源对 A股支持不稳定，之前 yfinance 就容易出现限流或数据不完整。

### 4.3 市场分析师分析技术面

市场分析师会看：

- 最近股价走势。
- 均线位置。
- MACD 动能。
- RSI 强弱。
- 布林带位置。
- 成交量和换手率。
- 是否接近支撑位或压力位。

它更像一个交易员或技术分析师，关注的是“价格现在处在什么状态”。

### 4.4 基本面分析师分析公司质量

基本面分析师会看：

- 公司属于什么行业。
- 估值是高还是低。
- ROE、ROA、ROIC 是否有吸引力。
- 毛利率、净利率是否稳定。
- 营收和利润是否增长。
- 资产负债率是否过高。
- 经营现金流是否支持利润。
- 三张表里有没有明显异常。

它更像一个价值投资研究员，关注的是“这家公司值不值得长期跟踪”。

### 4.5 多空双方辩论

系统里的激进分析师会尝试找上涨理由，例如：

- 估值有修复空间。
- 财务质量不错。
- 技术面出现突破。
- 行业环境改善。

保守分析师会尝试找风险，例如：

- 估值不便宜。
- 增长放缓。
- 现金流不足。
- 负债或应收账款压力。
- 技术面仍然弱。

中性分析师会尝试平衡双方观点，找出最关键的分歧。

这一步的价值不是让 AI 替你拍脑袋，而是帮你把“看多理由”和“看空理由”摆在同一张桌子上。

### 4.6 交易员形成行动建议

交易员会根据前面的讨论，给出类似下面的动作：

- Buy：买入。
- Hold：持有或观望。
- Sell：卖出或回避。

同时可能给出：

- 观察区间。
- 触发条件。
- 止损或风控位置。
- 观察周期。

对价值投资者来说，建议不要把这个动作当成指令，而要当成“当前证据下的初步结论”。

### 4.7 风控和组合经理复核

风险角色会检查：

- 下跌风险是否被低估。
- 波动是否太大。
- 当前是否适合重仓。
- 有没有必要等待更明确的信号。

组合经理最后会整合各方意见，输出最终的评级和投资摘要。

## 5. 如何阅读输出结果

一份完整输出通常包含：

- Rating：最终评级，例如 Buy、Hold、Sell。
- Executive Summary：一句话总结。
- Investment Thesis：投资逻辑。
- Bull Case：看多理由。
- Bear Case：看空理由。
- Risk-Reward：风险收益比。
- Price Target：目标价或观察价位。
- Time Horizon：观察周期。

你可以按下面顺序读：

1. 先看评级，但不要马上相信。
2. 再看它的核心理由是不是来自真实数据。
3. 接着看反方观点是否足够有力。
4. 最后看触发条件是否清楚。

如果一份报告只给结论，不讲条件，那它的参考价值就会下降。好的投研结论应该有“如果发生什么，就说明判断可能错了”。

## 6. 如何辩证看待它的结果

v3 的输出应该被当成“投研草稿”和“检查清单”，不应该被当成投资指令。

### 6.1 数据是真实的，但解释可能出错

Tushare 返回的数据是真实接口数据，但 AI 对数据的解释可能会犯错。例如：

- 把短期波动解释成长期趋势。
- 过度强调某个指标。
- 忽略行业特殊性。
- 对财务指标的因果关系判断过度自信。

所以你要看它有没有把结论和数据对应起来。

### 6.2 财报数据有滞后性

财务数据来自已经披露的报告期。比如当前是 5 月，最新财报可能还是一季度或上一年年报。

这意味着：

- 财报能说明过去经营质量。
- 财报不能直接代表今天的经营变化。
- 突发新闻、政策变化、订单变化可能没有被及时反映。

### 6.3 技术指标适合观察状态，不适合单独决策

MACD、RSI、均线这些指标适合回答“市场现在热不热、趋势强不强”。但它们不能单独回答“公司是否值得买”。

价值投资里，技术面更适合辅助择时和风控，而不是替代商业分析。

### 6.4 估值必须结合行业

不同公司不能只看 PE 高低。

例如：

- 银行股常看 PB、息差、资产质量。
- 消费股常看品牌、利润率、现金流。
- 科技股常看成长性、研发投入、周期位置。
- 周期股常看商品价格、产能和行业景气。

因此，如果 AI 只说“PE 低所以便宜”，你应该继续追问：为什么低？是被错杀，还是盈利正在下行？

### 6.5 目标价只能当作参考

系统可能输出 Price Target，但它不是保证会到达的价格。

你更应该关注：

- 它给目标价的理由是什么。
- 这个目标价对应什么估值假设。
- 如果业绩变差，目标价是否应该下修。
- 如果市场环境变差，安全边际是否足够。

## 7. 当前 v3 的边界

v3 已经比早期版本更适合 A股，但它还有明确边界。

当前已经加强的部分：

- A股代码自动走 Tushare。
- 支持日线行情。
- 支持常用技术指标。
- 支持估值和交易快照。
- 支持财务质量指标。
- 支持利润表、资产负债表、现金流量表。
- 支持主要指数和申万行业快照。
- 适合做单只股票的投研辅助。

当前还不适合的部分：

- 不适合直接实盘交易。
- 不提供自动下单。
- 不等于完整回测系统。
- 没有完整接入公告、新闻、研报、股东变化等信息。
- 没有分钟级高频数据分析。
- 没有自动覆盖全市场批量打分。
- 不能替代人工核对财报和公告。

如果后续继续升级，可以考虑加入：

- 公告和新闻数据。
- 行业对比表。
- 同行业估值分位。
- 历史估值分位。
- 股息和分红稳定性分析。
- 全市场批量初筛。
- 本地缓存，减少重复请求。
- 投研报告自动保存和对比。

## 8. 推荐使用方式

对投资爱好者来说，建议按“三步法”使用。

第一步，快速体检。

只开市场分析师和基本面分析师，看看系统对趋势、估值、财务质量的初步判断。

第二步，找分歧。

看多方和空方分别说什么。真正有价值的地方往往不是结论，而是双方争论的焦点。

第三步，人工复核。

打开公司公告、年报、季报、行业资料，验证 AI 提到的关键点是否准确。

你可以重点追问自己：

- 这家公司赚钱能力是否稳定？
- 现金流是否支持利润？
- 负债压力是否可控？
- 当前估值是否匹配成长性？
- 股价走势是否已经反映大部分利好？
- 如果判断错了，最可能错在哪里？

## 9. 给新用户的一句话总结

TradingAgents v3 不是“自动炒股机”，而是一个使用 Tushare A股数据和 DeepSeek 模型搭建的 AI 投研辅助框架。它可以帮你更快整理行情、估值、财务和风险线索，但它的输出必须被验证、被质疑、被放回你的投资框架里重新判断。
